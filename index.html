Cloud sync

Your word list, quiz progress, cached example sentences, and your Gemini API key all sync through your own free Firebase project, so the same list follows you across every device — no need to re-enter your key each time.

Checking…
Not synced yet? Set up your free Firebase project
  1. Go to console.firebase.google.com, sign in, and click Add project (the free Spark plan is enough).
  2. Inside the project, open Build → Firestore Database and click Create database (any region, start in "test mode" is fine — we'll lock it down in step 4).
  3. Click the </> (Web) icon on the project's home screen to register a web app, skip Hosting, and copy the firebaseConfig object it shows you.
  4. Open this file's source code, find the FIREBASE_CONFIG block near the top of the <script>, paste your values in, save, and re-deploy/re-upload the file wherever you're hosting it (GitHub Pages, etc). Every device that opens that same URL will now share one Firestore document.
  5. Back in Firestore, go to the Rules tab and replace them with the snippet below, then click Publish. This restricts access to only the one document this app uses:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /lexicard/main {
      allow read, write: if true;
    }
  }
}

Note on privacy: this rule keeps the data scoped to that single document path, but doesn't require a login — anyone who has your exact Firebase config and knows this app's document path could read or write it. That's fine for a personal project only you deploy, but don't publish your config values somewhere public search engines will index. If you'd rather lock it down with real authentication, ask me and I'll add sign-in.

Using Brave? Shields can sometimes clear storage for local (file://) pages. If your key keeps disappearing, click the Brave Shields icon (lion icon) in the address bar while this page is open and lower shields for this page, or check Settings → Shields → "Clear cookies and site data when you close all windows" isn't enabled for local files.
How to get a free key
  1. Go to aistudio.google.com/apikey and sign in with any Google account
  2. Click Create API key (no credit card needed)
  3. Copy the key (starts with AIza) and paste it above
  4. Important: Google now requires keys to have a restriction set. On the key's page, click Edit → under API restrictions choose Restrict key → select Generative Language API → Save. Unrestricted keys will stop working.
Free tier: 1,500 requests/day, 10/minute on Gemini 2.5 Flash — more than enough for quiz sessions. No spending occurs; once the daily limit is hit, requests pause until it resets.
